# Fix Image Upload Issue - Plan

## Problem
- Admin shows demo images (Unsplash) instead of user's uploaded images
- Every image upload shows "upload failed"
- Current issue: Base64 in-memory storage is not persistent

## Solution
Implement Supabase Storage for image uploads. The app already has Supabase credentials configured.

## Steps

### Step 1: Create Storage Bucket in Supabase
1. Go to your Supabase dashboard → Storage
2. Create new bucket named `uploads`
3. Make it **public** (for image access)
4. Configure RLS policy to allow public read access:
   ```sql
   -- Allow public read access
   CREATE POLICY "Public Access" ON storage.objects
   FOR SELECT USING (bucket_id = 'uploads');
   
   -- Allow authenticated uploads (or public for admin)
   CREATE POLICY "Admin Upload" ON storage.objects
   FOR INSERT WITH CHECK (bucket_id = 'uploads');
   ```

### Step 2: Update Upload API
Modify `/src/app/api/upload/route.ts`:
- Add Supabase storage client
- Upload file to `uploads` bucket instead of base64 in-memory
- Return the public URL of uploaded image

### Step 3: Test Upload
- Try uploading an image in admin panel
- Verify image URL is returned and saved
